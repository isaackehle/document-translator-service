# Documentation Guidelines

This document outlines the conventions for organizing and managing documentation in this repository.

## Document Organization

All new documentation files should be added to the `docs/` folder. This includes:
- Configuration guides
- Setup instructions
- Technical documentation
- Process descriptions

## File Naming Conventions

- Use descriptive names that clearly indicate the document's purpose
- Use kebab-case for multi-word filenames (e.g., `ai-tools-configuration.md`)
- Keep filenames concise but meaningful
- All filenames must be lowercase

## Document Structure

All documentation should follow the standard format:
1. YAML frontmatter with appropriate tags
2. Clear H1 heading with tool icon when applicable
3. Standard sections: Installation, Configuration, Usage, References
